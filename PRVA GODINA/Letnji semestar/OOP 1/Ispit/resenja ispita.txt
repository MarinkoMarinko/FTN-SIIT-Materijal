1 int iterativeFact(int broj) {
    int rezultat = 1;
    for(int i = broj; i > 1; i--) {
        rezultat *= i;
    }
    return rezultat;    
}

public int recursiveFact(int broj) {
    if(broj == 1){
        return 1;
    }
    return broj * recursiveFact(broj - 1);
}

3 public class Osoba {
    private String ime;

    // Konstruktor
    public Osoba(String ime) {
        this.ime = ime;
    }

    // Metoda koja dodaje tekst na kraj imena
    public void dodaj(String txt) {
        ime = ime + txt;
    }

    // Getter
    public String getIme() {
        return ime;
    }

    // Glavni program za testiranje
    public static void main(String[] args) {
        Osoba osoba = new Osoba("Marko");
        osoba.dodaj(" Petrović");

        System.out.println(osoba.getIme());
    }
}

        super(ime);
    }

    public void dodaj(String txt) {
        ime = txt + ime;

Tokovi (streams) – U Javi služe za ulaz i izlaz podataka. Postoje bajtovski tokovi (InputStream, OutputStream) i karakter tokovi (Reader, Writer). Koriste se za rad sa fajlovima, mrežom itd.

Gde se čuva niz? – Niz (array) se uvek čuva u heap memoriji, jer je to objekat u Javi.

Garbage collector kada se poziva? – Automatski ga poziva JVM kada objekat više nema nijednu referencu. Ne može se tačno kontrolisati kada će se desiti.

Overriding – Kada podklasa redefiniše metodu iz nadklase sa istim potpisom (ime, parametri). Koristi se za specifično ponašanje.

Kako se spajaju stringovi? – Najčešće operatorom +, npr. "A" + "B", ili metodom concat().

Apstraktne klase – šta imaju? – Mogu imati obične i apstraktne metode. Ne mogu se instancirati, služe kao osnova za druge klase.

Nakon kompajliranja šta se dobija? – Dobija se .class fajl koji sadrži bytecode (izvršava ga JVM).

Flow Layout – Raspoređuje komponente redom: levo → desno, pa novi red ako nema mesta.

GUI ActionListener – Interfejs koji reaguje na akcije korisnika (npr. klik dugmeta). Implementira se metoda actionPerformed().

Da li konstruktor može imati return? – Ne. Konstruktor nema povratni tip i ne koristi return kao metode.

Nasleđivanje – Java dozvoljava samo jednostruko nasleđivanje klasa, ali više interfejsa može da se implementira.

Referenca i objekti – Objekti se nalaze u heap memoriji, a reference (pokazivači na njih) u stack memoriji.

Java prenos parametara – Uvek je po vrednosti. Kod objekata se kopira vrednost reference, ne sam objekat.

Look and Feel – Određuje vizuelni stil GUI aplikacije (npr. Windows, Nimbus izgled).

Singleton – Dizajn obrazac koji omogućava da klasa ima samo jednu instancu u celom programu.

Kako se spajaju stringovi? – Operator + ili concat() (isti princip kao gore, ponovljeno pitanje).

Gde se čuva niz? – U heap memoriji (ponovljeno pitanje).

Garbage collector kada se aktivira? – Automatski, kada JVM proceni da treba osloboditi memoriju (ponovljeno pitanje).

Da li se parametri šalju po vrednosti ili referenci ili pointeru? – U Javi se sve šalje po vrednosti, čak i reference na objekte.
